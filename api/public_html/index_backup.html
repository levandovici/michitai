📊 Database Schema

Core Tables

users

User accounts and authentication

  • id - Primary key
  • email - Unique email address
  • password_hash - bcrypt hashed password
  • api_token - Authentication token
  • plan_type - Free/Standard/Pro
  • memory_used_mb - Current memory usage
games

Game configurations and logic

  • id - Primary key
  • user_id - Owner reference
  • name - Game name
  • json_structure - Puzzle logic JSON
  • json_properties - Game properties
  • memory_used_mb - Memory consumption
players

Player data and progress

  • id - Unique player ID
  • game_id - Game reference
  • password_guid - Player password
  • json_data - Player data with types
  • memory_used_mb - Memory usage

Supporting Tables

subscriptions

Payment and subscription data

  • paypal_subscription_id - PayPal reference
  • plan_type - Subscription level
  • status - Active/Cancelled/Suspended
  • next_billing_date - Next payment
rooms

Game rooms and sessions

  • id - Room identifier
  • game_id - Associated game
  • max_players - Player limit
  • is_private - Privacy setting
notifications

Email and Slack notifications

  • type - email/slack
  • recipient - Target address
  • subject - Message subject
  • status - Delivery status

🔗 API Endpoints

Authentication

POST /api/register - Create new user account
POST /api/login - Authenticate user
GET /api/user - Get user profile

Game Management

POST /api/game/create - Create new game
GET /api/game/get/{id} - Get game details
PUT /api/game/update - Update game configuration
DELETE /api/game/delete/{id} - Delete game

Player Management

POST /api/player/create - Create new player
POST /api/player/auth - Authenticate player
PUT /api/player/update - Update player data
GET /api/player/get/{id} - Get player data

Subscriptions & Payments

POST /api/subscribe - Create subscription
GET /api/subscription/status - Get subscription status
POST /api/subscription/cancel - Cancel subscription
POST /api/webhook/paypal - PayPal webhook handler

Monitoring & Stats

GET /api/monitor/user - User statistics
GET /api/monitor/system - System statistics
