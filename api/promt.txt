Create a multiplayer API web constructor hosted on Hostinger using PHP 8.x, HTML, JavaScript (ES6+), CSS (Tailwind CSS), MySQL 8.x, and cron jobs. The system must support user management, game configuration, player data, puzzle logic constructor, and subscription-based payments via PayPal (primary, supporting global Visa/Mastercard with payouts to a MAIB card in Moldova via SWIFT: AGRNMD2X) with fallback to Paynet or manual MAIB bank transfers. Enforce system limits (200 GB storage, 200 users) and plan limits (Free: 250 MB, 80 users, 1 game, 100 players, 5 rooms, 1,000 API calls/day; Standard: 1 GB, 80 users, 1 game, 1,000 players, 50 rooms, 5 communities, 1,000 messages/day, 10,000 API calls/day, ~$50/month; Pro: 2.5 GB, 40 users, 3 games, 2,500 players, unlimited rooms/communities/chats, 1M API calls/day, ~$200/month). Ensure data type compatibility with JavaScript, PHP, and C# (Boolean, Char, Byte, Short, Integer, Long, Float, Double, String, Array, Enum). Send notifications for limits and subscription events via email (Free/Standard) or Slack (Pro).

**Requirements**:
1. **Database Schema** (MySQL, database: `multiplayer_api`):
   - `users`: `user_id` (int, PK), `email` (varchar), `password` (varchar, hashed), `api_token` (varchar, UUIDv4), `plan_type` (enum: Free, Standard, Pro), `memory_used_mb` (decimal), `memory_limit_mb` (decimal), `api_calls_today` (int), `paypal_customer_id` (varchar).
   - `subscriptions`: `subscription_id` (int, PK), `user_id` (int, FK), `paypal_subscription_id` (varchar), `plan_type` (enum), `status` (enum: active, canceled, past_due), `created_at` (timestamp).
   - `games`: `game_id` (int, PK), `user_id` (int, FK), `json_structure` (json), `json_properties` (json), `json_rooms` (json), `json_communities` (json), `json_chats` (json).
   - `players`: `player_id` (varchar, GUID), `game_id` (int, FK), `password_guid` (varchar, GUID), `json_data` (json).
   - `rooms`: `room_id` (int, PK), `game_id` (int, FK), `json_data` (json).
   - `communities`: `community_id` (int, PK), `game_id` (int, FK), `json_data` (json), `privileges` (json).
   - `chats`: `chat_id` (int, PK), `game_id` (int, FK), `json_messages` (json).
   - `triggers`: `trigger_id` (int, PK), `game_id` (int, FK), `name` (varchar), `parameters` (json).
   - `timers`: `timer_id` (int, PK), `game_id` (int, FK), `player_id` (varchar, FK), `value` (float), `multiplier` (float), `trigger_id` (int, FK).

2. **Frontend** (HTML, JavaScript, Tailwind CSS):
   - Single-page app with forms for game configuration and drag-and-drop puzzle logic constructor.
   - Use CDNs for jQuery, Tailwind CSS, PayPal JavaScript SDK.
   - Puzzle logic: Support triggers (string, JSON params), timers (set, math, multipliers), logic (If, If-Else, For, Switch, While, Do-While), operators (+, -, *, /, %), functions (Power, Sqrt, Qrt, Random), logic operators (<, >, <=, >=, ==, !=, !), and data types (Boolean, Char, Byte, Short, Integer, Long, Float, Double, String, Array, Enum).
   - Manage matchmaking, rooms, communities, chats (create, join, data, remove, leave, write, edit, read).
   - Payment integration: Use PayPal JavaScript SDK for card input or redirect to PayPal/Paynet.

3. **Backend** (PHP, REST API):
   - Use PDO for MySQL, JSON responses.
   - Endpoints:
     - `/api/register`: POST (email, password) → Create user, API token.
     - `/api/login`: POST (email, password) → Return API token.
     - `/api/subscribe`: POST (api_token, plan_type) → Redirect to PayPal/Paynet, create subscription.
     - `/api/subscription/update`: POST (api_token, plan_type) → Update subscription.
     - `/api/subscription/cancel`: POST (api_token) → Cancel subscription.
     - `/api/game/create`: POST (api_token, json_structure, json_properties) → Create game, check memory.
     - `/api/player/create`: POST (api_token, game_id) → Generate GUIDs for `player_id`, `password_guid`.
     - `/api/player/update`: POST (player_id, json_data) → Update progress, check memory.
     - `/api/room/{action}`, `/api/community/{action}`, `/api/chat/{action}`, `/api/matchmaking/{action}`: POST (create, join, data, remove, leave, write, edit, read, privileges).
     - `/api/trigger`: POST (name, parameters) → Store trigger.
     - `/api/timer/{action}`: POST (set, math, multiplier) → Manage timers.
     - `/api/monitor/user`: GET (user_id) → Return memory usage.
     - `/api/monitor/system`: GET → Return total memory, users.
   - Memory management: Enforce plan limits (250/1000/2500 MB), system limit (204,800 MB).
   - API call limits: Track `api_calls_today`, reset via cron.

4. **Payment Integration**:
   - Primary: PayPal (install `paypal/rest-api-sdk-php` via Composer, use API credentials, handle webhooks for subscription events, configure payouts to MAIB card via SWIFT: AGRNMD2X).
   - Fallback: Paynet (Moldova-based, redirect to payment page, manual recurring billing via cron if needed).
   - Manual fallback: MAIB bank transfers (verify via MPay, update subscriptions via cron).
   - Ensure PCI compliance with PayPal/Paynet hosted checkout, use HTTPS.

5. **Cron Jobs**:
   - Daily: Reset `api_calls_today` (00:00 UTC).
   - Every minute: Update `timers`, trigger on completion.
   - Daily: Check PayPal/Paynet subscription status (e.g., past_due).
   - Every 5 minutes: Send queued notifications (email via PHPMailer for Free/Standard, Slack webhook for Pro).

6. **Security**:
   - Hash passwords with bcrypt.
   - Use PDO prepared statements.
   - Authenticate API calls with `api_token`.
   - Validate JSON inputs for C# compatibility.
   - Use HTTPS for all endpoints.

7. **Testing**:
   - Test registration, login, subscription creation (PayPal/Paynet), game/player management, puzzle constructor (all logic, JSON output).
   - Verify memory/API call limits, subscription events, notifications, cron jobs.
   - Confirm payouts to MAIB card (SWIFT: AGRNMD2X).
   - Ensure data type compatibility (JavaScript/PHP/C#).

8. **Deployment**:
   - Hostinger: Configure PHP, MySQL, cron jobs, SSL.
   - Set up PayPal (API credentials, webhooks), Paynet (if used), PHPMailer (SMTP), Slack webhook.
   - Provide MAIB card details for payouts.
   - Back up database regularly.

**Deliverables**:
- Full source code for frontend (HTML, JavaScript, Tailwind CSS) and backend (PHP, MySQL schema).
- Configuration for PayPal/Paynet, cron jobs, PHPMailer, Slack webhook.
- Documentation for deployment on Hostinger and testing steps.
