### Programming Plan for Multiplayer API Web Constructor with Moldova Payment System

**Objective**: Build a web-based constructor for a multiplayer API on Hostinger, using PHP, HTML, JavaScript, CSS, MySQL, and cron jobs. Support user management, game configuration, player data, backend logic, puzzle logic constructor, and subscription-based payments via a gateway allowing global credit card payments (Visa/Mastercard) with funds received on a MAIB card in Moldova. Enforce memory limits (200 GB, 200 users) and send notifications. Ensure data type compatibility with JavaScript, PHP, and C#.

#### 1. System Architecture
- **Hosting**: Hostinger web server (PHP 8.x, MySQL 8.x, cron job support).
- **Frontend**: HTML, JavaScript (ES6+), CSS (Tailwind CSS).
- **Backend**: PHP (PDO for MySQL, REST API endpoints).
- **Database**: MySQL (store users, games, players, rooms, communities, chats, subscriptions).
- **Payment System**: PayPal for global credit card payments (Visa/Mastercard) with payouts to your MAIB card; fallback to Paynet (Moldova-compatible) or manual MAIB bank transfers.
- **Cron Jobs**: Reset API calls, process timers, manage subscriptions, send notifications.
- **Data Types**: Compatible with JavaScript/PHP/C#:
  - Boolean (bool), Char (string(1)), Byte (int8), Short (int16), Integer (int32), Long (int64), Float (float), Double (double), String (string), Array (array/list), Enum (enum).

#### 2. Database Schema
- **Database**: `multiplayer_api`
- **Tables**:
  - **users**: `user_id` (int, PK), `email` (varchar), `password` (varchar, hashed), `api_token` (varchar, UUIDv4), `plan_type` (enum: Free, Standard, Pro), `memory_used_mb` (decimal), `memory_limit_mb` (decimal), `api_calls_today` (int), `paypal_customer_id` (varchar).
  - **subscriptions**: `subscription_id` (int, PK), `user_id` (int, FK), `paypal_subscription_id` (varchar), `plan_type` (enum), `status` (enum: active, canceled, past_due), `created_at` (timestamp).
  - **games**: `game_id` (int, PK), `user_id` (int, FK), `json_structure` (json), `json_properties` (json), `json_rooms` (json), `json_communities` (json), `json_chats` (json).
  - **players**: `player_id` (varchar, GUID), `game_id` (int, FK), `password_guid` (varchar, GUID), `json_data` (json).
  - **rooms**: `room_id` (int, PK), `game_id` (int, FK), `json_data` (json).
  - **communities**: `community_id` (int, PK), `game_id` (int, FK), `json_data` (json), `privileges` (json).
  - **chats**: `chat_id` (int, PK), `game_id` (int, FK), `json_messages` (json).
  - **triggers**: `trigger_id` (int, PK), `game_id` (int, FK), `name` (varchar), `parameters` (json).
  - **timers**: `timer_id` (int, PK), `game_id` (int, FK), `player_id` (varchar, FK), `value` (float), `multiplier` (float), `trigger_id` (int, FK).

#### 3. Plan Limits
- **Free**: 250 MB, 80 users, 1 game, 100 players, 5 rooms, 1,000 API calls/day, no subscription.
- **Standard**: 1 GB, 80 users, 1 game, 1,000 players, 50 rooms, 5 communities, 1,000 messages/day, 10,000 API calls/day, ~$50/month/user.
- **Pro**: 2.5 GB, 40 users, 3 games, 2,500 players, unlimited rooms/communities/chats, 1M API calls/day, ~$200/month/user.
- **System**: 200 GB (204,800 MB), 200 users.

#### 4. Payment System Integration
- **Primary Option**: PayPal
  - **Rationale**: PayPal supports global credit card payments (Visa/Mastercard) and payouts to MAIB cards in Moldova via bank transfer (SWIFT: AGRNMD2X). Widely used, supports recurring billing.[](https://www.moneyto.co.uk/moldova)[](https://wise.com/us/swift-codes/AGRNMD2XXXX)
  - **Setup**: Create PayPal business account, obtain API credentials (client ID, secret). Install PayPal PHP SDK (`composer require paypal/rest-api-sdk-php`).
  - **Subscription Flow**:
    - User selects Standard/Pro plan via constructor.
    - Frontend: Use PayPal JavaScript SDK for card input (PCI-compliant) or redirect to PayPal.
    - Backend: Create PayPal billing plan/agreement for Standard (~$50/month) or Pro (~$200/month), store `paypal_subscription_id` in `subscriptions`.
    - Handle PayPal webhooks (e.g., `BILLING.SUBSCRIPTION.ACTIVATED`, `PAYMENT.SALE.COMPLETED`, `BILLING.SUBSCRIPTION.CANCELLED`).
    - Payouts: Configure PayPal to transfer funds to your MAIB card (SWIFT: AGRNMD2X, account details required).
  - **Features**:
    - Recurring billing: Monthly charges via PayPal.
    - Card payments: Global Visa/Mastercard support.
    - Retry logic: PayPal handles failed payments.
    - Upgrade/Downgrade: Update plan, sync with PayPal.
    - Cancellation: Cancel subscription, update `status`.
- **Fallback Option**: Paynet
  - **Rationale**: Moldova-based payment provider, supports card payments, integrates with MAIB.[](https://www.moneyto.co.uk/moldova)
  - **Setup**: Contact Paynet for API access, obtain credentials.
  - **Flow**: Redirect to Paynet’s payment page, store transaction IDs, manual recurring billing via cron (if Paynet lacks subscription support).
  - **Payouts**: Funds received directly to your MAIB card.
- **Manual Fallback**: MAIB Bank Transfers
  - Prompt users for monthly bank transfers to your MAIB account (SWIFT: AGRNMD2X).
  - Verify payments manually or via MAIB’s MPay service, update `subscriptions` via cron.
- **Security**: Use PayPal’s hosted checkout or Paynet’s payment page for PCI compliance. HTTPS for all transactions.

#### 5. Data Flows
- **User**:
  - Register: Email, password (hashed), API token (UUIDv4), `paypal_customer_id` (for Standard/Pro).
  - Login: Authenticate, return API token.
  - Memory: Track `memory_used_mb`, enforce plan limits.
- **Game**:
  - Create: JSON structure/properties via constructor, store in `games`.
  - Rooms: Store match data in `json_rooms`.
  - Communities: Store in `json_communities`.
  - Chats: Store messages in `json_chats`.
- **Player**:
  - ID: GUID (server-generated, C# Guid-compatible).
  - Password: GUID (stored locally or on game services: Google Play, Apple Store, etc.).
  - Progress: Store in `json_data`.
- **Backend**:
  - Triggers: Store in `triggers`, execute via cron/API.
  - Timers: Store in `timers`, process via cron, trigger on completion.
  - Notifications: Email (Free/Standard), email/Slack (Pro).
  - Matchmaking: Store in `rooms`, manage via endpoints.
  - Rooms/Communities/Chats: Store JSON, manage via endpoints.

#### 6. Web Constructor (Frontend)
- **HTML**: Single-page app, forms for game config, drag-and-drop puzzle constructor.
- **JavaScript**:
  - Use CDN: jQuery, Tailwind CSS, PayPal JavaScript SDK.
  - Build JSON structure/properties via forms.
  - Puzzle Logic Constructor:
    - Main: Trigger (string, JSON params), Timer (set, math, multipliers).
    - Data: Fetch JSON (player, game, matchmaking, rooms, communities, chats).
    - Structures: Boolean, Char, Byte, Short, Integer, Long, Float, Double, String, Array, Enum.
    - Logic: If, If-Else, For, Switch, While, Do-While.
    - Operators: +, -, *, /, %.
    - Functions: Power, Sqrt, Qrt, Random.
    - Logic Operators: <, >, <=, >=, ==, !=, !.
    - Notifications: Player device, email, push.
    - Matchmaking: Create, Find, Data, Remove, Redirect, Leave.
    - Rooms: Create, Join, Data, Remove, Leave.
    - Chats: Write, Edit, Read.
  - Payment: Use PayPal SDK for card input or redirect to PayPal/Paynet.
- **CSS**: Tailwind CSS for responsive UI.
- **Data Types**: JSON output compatible with JavaScript (objects, arrays), PHP (arrays, strings), C# (classes, lists, enums).

#### 7. Backend (PHP)
- **Endpoints** (REST API, JSON responses):
  - `/api/register`: POST (email, password) → Create user, API token.
  - `/api/login`: POST (email, password) → Return API token.
  - `/api/subscribe`: POST (api_token, plan_type) → Redirect to PayPal/Paynet, create subscription, update `users`/`subscriptions`.
  - `/api/subscription/update`: POST (api_token, plan_type) → Update subscription.
  - `/api/subscription/cancel`: POST (api_token) → Cancel subscription.
  - `/api/game/create`: POST (api_token, json_structure, json_properties) → Create game, check memory.
  - `/api/player/create`: POST (api_token, game_id) → Generate GUIDs for `player_id`, `password_guid`.
  - `/api/player/update`: POST (player_id, json_data) → Update progress, check memory.
  - `/api/room/{action}`: POST (create, join, data, remove, leave) → Manage rooms.
  - `/api/community/{action}`: POST (create, join, leave, privileges) → Manage communities.
  - `/api/chat/{action}`: POST (write, edit, read) → Manage chats.
  - `/api/trigger`: POST (name, parameters) → Store trigger.
  - `/api/timer/{action}`: POST (set, math, multiplier) → Manage timers.
  - `/api/matchmaking/{action}`: POST (create, find, data, remove, redirect, leave) → Manage matchmaking.
  - `/api/monitor/user`: GET (user_id) → Return memory usage, remaining.
  - `/api/monitor/system`: GET → Return total memory, users.
- **Memory Management**:
  - Check `memory_used_mb` vs. `memory_limit_mb` (250/1000/2500 MB).
  - Check total `memory_used_mb` vs. 204,800 MB.
  - Update `memory_used_mb` on game/player/room/community/chat creation.
- **API Call Limits**: Track `api_calls_today`, reset via cron, enforce plan limits.

#### 8. Cron Jobs
- **Daily Reset**: Reset `api_calls_today` (00:00 UTC).
- **Timer Processing**: Update `timers`, trigger on completion (every minute).
- **Subscription Check**: Verify PayPal/Paynet payment status (e.g., past_due), daily.
- **Notifications**: Send queued notifications (email/Slack, every 5 minutes).

#### 9. Notifications
- **Triggers**: User memory limit, system 200 GB limit, plan user limit (80 Free/Standard, 40 Pro), API call limits, subscription events (failed payment, cancellation).
- **Methods**: Email (PHPMailer for Free/Standard), Slack (webhook for Pro).
- **Log**: Store in `notifications.log`.

#### 10. Security
- Hash passwords (bcrypt).
- Use PDO prepared statements.
- Authenticate API calls with `api_token`.
- Validate JSON inputs (C# compatibility).
- Use PayPal’s hosted checkout or Paynet’s payment page for PCI compliance.
- HTTPS for all endpoints.

#### 11. Testing

- Test puzzle constructor (all puzzles, JSON output).
- Test memory/API call limits, subscription events, notifications.
- Test cron jobs (reset, timers, subscriptions, notifications).
- Verify payouts to MAIB card (SWIFT: AGRNMD2X).
- Verify data type compatibility (JavaScript/PHP/C#).

#### 12. Deployment
- Hostinger: Set up PHP, MySQL, cron jobs, SSL.
- Configure PayPal (API credentials, webhooks), Paynet (if used), PHPMailer (SMTP), Slack webhook.
- Provide MAIB card details (SWIFT: AGRNMD2X) for PayPal payouts.
- Back up database regularly.

#### 13. Workflow
1. User registers, gets API token.
2. User subscribes (Standard/Pro) via PayPal (card payment) or Paynet, funds sent to your MAIB card.
3. User creates game via constructor, stores JSON (structure, properties, rooms, communities, chats).
4. Players join (GUID-based), store progress.
5. Backend manages triggers, timers, matchmaking, rooms, communities, chats.
6. Monitor memory/API calls/subscriptions, notify on limits via email/Slack.

#### Notes
- PayPal is primary due to global card support and MAIB payouts via SWIFT.[](https://www.moneyto.co.uk/moldova)[](https://wise.com/us/swift-codes/AGRNMD2XXXX)

- Manual MAIB transfers as last resort (verify via MPay).
- Optimize with indexes (`user_id`, `game_id`, `player_id`).
- Extend for analytics (Pro plan) with cron-based reports.
- Test edge cases (e.g., failed payments, invalid JSON, exceeding limits).
